¿Qué es GitHub Local?

Gestiona tus proyectos desde tu computadora con Git y sincronízalos con GitHub para facilitar el trabajo colaborativo y el control de versiones.

Definición

GitHub Local se refiere al uso de Git en tu máquina local para gestionar versiones de tus proyectos, con la posibilidad de conectarlo a GitHub como repositorio remoto. Esto te permite mantener un historial completo de tus cambios, colaborar con otros programadores, y restaurar versiones anteriores cuando lo necesites.

El trabajo local con Git es esencial para los desarrolladores, ya que proporciona una forma segura y flexible de gestionar el código fuente sin depender siempre de una conexión a Internet. GitHub entra en juego como el medio para compartir, colaborar y almacenar copias remotas de estos repositorios.

Pasos básicos para trabajar con GitHub Local

  1. 1. Instala Git: Descarga e instala Git desde su página oficial (git-scm.com). Es una herramienta esencial para el control de versiones. Al instalarlo, también obtienes Git Bash, una terminal con comandos de Git listos para usarse.
  2. 2. Configura tu identidad: Antes de empezar a trabajar, necesitas identificarte para que Git registre quién realiza los cambios. Ejecuta los siguientes comandos en Git Bash: 
    git config --global user.name "Tu Nombre"
    git config --global user.email "tucorreo@ejemplo.com"
    Esto asegura que cada commit (registro de cambio) esté correctamente atribuido a ti.
  3. 3. Inicia un repositorio o clona uno existente: Puedes comenzar un proyecto desde cero con: 
    git init
    Esto crea un repositorio local en la carpeta actual.
    O si ya tienes un proyecto en GitHub, clónalo con: 
    git clone https://github.com/usuario/repositorio.git
    Esto descarga una copia del repositorio remoto en tu computadora.
  4. 4. Agrega y confirma tus cambios: Cada vez que hagas modificaciones en tu proyecto, debes agregarlas y confirmarlas (committearlas) para que queden registradas: 
    git add .
    git commit -m "Descripción de los cambios"
    El primer comando selecciona los archivos a rastrear, y el segundo guarda un registro con un mensaje explicativo.
  5. 5. Conecta con GitHub y sincroniza: Si creaste un repositorio nuevo en GitHub, puedes conectarlo con tu repositorio local: 
    git remote add origin https://github.com/usuario/repositorio.git
    Luego puedes subir tus cambios con: 
    git push -u origin main
    Esto envía tus commits locales al servidor remoto en GitHub.
